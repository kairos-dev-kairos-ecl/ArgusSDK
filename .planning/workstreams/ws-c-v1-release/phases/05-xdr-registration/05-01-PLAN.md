<objective>
Implement the real XDR Mode-1 HTTP client in internal/auth: a remoteRegistrar (auth.Registrar) that
exchanges GroupID+InstallToken for a server-assigned InstanceID+Credential via
POST /api/v1/sdk/register, and a remoteCredentialRefresher (auth.CredentialRefresher) that rotates the
credential via POST /api/v1/sdk/credential/refresh. Both speak the contract documented in
internal/auth/auth.go over a stdlib net/http client pinned to TLS 1.3 (no new dependency). Build an
in-process httptest mock XDR server and a unit suite covering register, refresh, install-token
single-use invalidation, and failure paths (non-2xx, transport error → typed error, no panic, no
partial state).

/api/v1/sdk/credential/refresh), R-55 (mock-XDR-backed tests for register/refresh/token-reuse/failures).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| agent → XDR registration endpoint | InstallToken (one-time secret) + GroupID cross the network to obtain InstanceID + Credential |
| agent → XDR refresh endpoint | InstanceID + current Credential cross the network to obtain a rotated Credential |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Information disclosure | TLS downgrade on the registration channel | mitigate | http.Client.Transport.TLSClientConfig comes from connector.NewTLSConfig (MinVersion TLS 1.3); InsecureSkipVerify is never set (grep gate) — no cleartext or downgraded path exists |
| T-05-02 | Spoofing / replay | install-token theft & reuse | mitigate | Single-use enforced server-side (mock returns 409/401 on reuse); client maps that to ErrInstallTokenConsumed and returns nil response so a stolen-but-consumed token cannot mint a second identity |
| T-05-03 | Spoofing | registration response forgery (rogue XDR) | mitigate | Server identity verified by TLS 1.3 cert validation (no InsecureSkipVerify); the agent trusts only certs chaining to the configured/system roots |
| T-05-04 | Repudiation | silent failure on non-2xx / transport error | mitigate | Non-2xx → typed httpStatusError, transport failure → wrapped error; Register/Refresh return (nil/"", err) with no partial state — failures are never swallowed |
| T-05-SC | Tampering | dependency supply chain | mitigate | No new module added; client uses stdlib net/http only — verified by the go.mod no-new-module acceptance gate |
</threat_model>
