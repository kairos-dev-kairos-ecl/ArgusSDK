<objective>
Replace the Linux stub in internal/collector/euc/linux.go with a real eBPF-based Shadow-AI
OSCollector, following 06-RESEARCH.md: github.com/cilium/ebpf v0.21.0 (pure-Go, no cgo) loading a
PRECOMPILED, COMMITTED eBPF object produced by bpf2go. A kprobe on tcp_v4_connect and
tcp_v6_connect surfaces outbound connect events; a ringbuf delivers them to userspace, where they
are filtered against cfg.AIEndpoints (remote host) and cfg.LocalInferencePorts (local port) and
mapped to euc.Observation. The gopsutil/v4 connection-table path provides the no-privilege
local-inference detection so the collector still produces euc.local_inference signals when the
eBPF path degrades.

(matchHost / matchPort / local-inference-port helper). This plan CONSUMES those helpers from package
euc; it does NOT create or fall back to creating netcommon.go.

privileges), SC-4 (config-driven watch lists), SC-5 (per-platform tests that t.Skip elsewhere),
SC-6 (low-privilege contract preserved). R-61, R-64, R-65.

Output: real linux.go + linux_test.go + committed bpf2go artifacts in package euc + go.mod update for
cilium/ebpf. NO change to euc.go, euc_noop.go, Observation, Config, fanOut, or netcommon.go (owned by
06-00).

Scope fence: the agent must NOT require clang or cgo at build or runtime — the .o is compiled in

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| kernel connect events → linux.go userspace | Decoded eBPF event fields (remote addr/host, port, PID, comm) are untrusted input |
| cfg watch lists → filter | AIEndpoints / LocalInferencePorts come from agent config |
| new module (cilium/ebpf) → build graph | Third-party dependency entering the Linux build only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Elevation of Privilege | eBPF load / kprobe attach | mitigate | Run with CAP_BPF,CAP_PERFMON (not root, no SYS_ADMIN escalation); on EPERM/EACCES degrade to no-op via Start returning nil — never request more privilege (Pitfall 2, ASVS V4) |
| T-06-02 | Information Disclosure | over-capture beyond Shadow-AI scope | mitigate | Capture connection METADATA only (5-tuple at connect); no payload, no general process enumeration; ProcessName only from the connecting task's own comm/PID (Pitfall 5) |
| T-06-03 | Tampering | decoded eBPF event fields | mitigate | Treat remote host/PID/comm as untrusted: bound string lengths, ignore malformed events, normalize host before match (ASVS V5) |
| T-06-04 | Tampering | committed eBPF object (tcpconnect_bpfel.o) | mitigate | Object regenerated only in CI/maintainer context from committed tcpconnect.c; reviewers diff the .o on regeneration; agent never compiles it |
| T-06-05 | Denial of Service | capability-denied path | mitigate | gopsutil local-port path needs no caps and keeps euc.local_inference flowing; Start never errors out the agent on capture-init failure |
| T-06-SC | Tampering | cilium/ebpf install (supply chain) | mitigate | Blocking human-verify checkpoint (Task 1) confirms v0.21.0 on pkg.go.dev before go get; pure-Go, no cgo |
</threat_model>
