<objective>
Replace the macOS stub in internal/collector/euc/darwin.go with a real, SHIPPABLE Shadow-AI
OSCollector. Per 06-RESEARCH.md (Pitfall 4), a real NEDNSProxyProvider Network Extension cannot
ship in a `go install`'d CLI — it needs a signed `.app` bundle, the managed
com.apple.developer.networking.networkextension entitlement (Apple-approved), Developer-ID signing,
notarization, and OSSystemExtensionRequest activation. None of that is satisfiable in this phase.

The shippable v1.0 darwin implementation is therefore a NO-ROOT established-connection sampler
(gopsutil/v4 net.Connections): poll established outbound connections + listening sockets, match
against cfg.AIEndpoints / cfg.LocalInferencePorts, emit euc.Observation. It honors the low-privilege
contract, ships as a CLI today, and produces the same Observation shape. The full Network Extension
is EXPLICITLY DEFERRED to a future packaging/signing phase.

This plan depends on 06-00 (Wave 0), which is the SINGLE owner of the tag-free netcommon.go helpers
(matchHost / matchPort / local-inference-port helper). This plan CONSUMES those helpers from package
euc; it does NOT create or fall back to creating netcommon.go.

Purpose: SC-3 (darwin observer forwarding Observations), SC-4 (config-driven watch lists), SC-5
(per-platform tests that t.Skip elsewhere), SC-6 (low-privilege contract preserved). R-63, R-64, R-65.

Output: real darwin.go (sampler) + darwin_test.go. NO new module (gopsutil/v4 already present). NO
change to euc.go, euc_noop.go, Observation, Config, fanOut, or netcommon.go (owned by 06-00).

SCOPE BOUNDARY (explicit): this plan implements the sampler ONLY. The full Network Extension
(packaging, signing, entitlements, notarization, system-extension activation) is OUT OF SCOPE for
this phase and is a separate later packaging phase. Any task referencing entitlements/.app/signing
inside this phase is a descope signal (Pitfall 4).
