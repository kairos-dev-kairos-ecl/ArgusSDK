<design_note>
newOSCollector is UNEXPORTED and defined once per build tag, so it cannot be referenced from package
agent directly. Add ONE exported wrapper in package euc:

  // oscollector.go — NO build tag (compiled on every GOOS, imports nothing platform-specific).
  func NewOSCollector(cfg Config) OSCollector {
      impl := newOSCollector(cfg)        // build-tag-selected per-platform impl
      if impl == nil {                   // belt-and-suspenders fallback
          return NewNoopOSCollector()
      }
      return impl
  }

oscollector.go has NO build tag and imports nothing platform-specific — it only calls the
tag-selected newOSCollector and the existing noop, so it cannot drag a heavy dep into other GOOS
builds (Pitfall 6). The per-platform newOSCollector already returns a real impl that degrades to
emitting nothing when capability is absent, so the noop fallback here is the safety net for any
future build target without a newOSCollector. Confirm every GOOS in scope (linux/windows/darwin)
defines newOSCollector so the wrapper compiles everywhere; if a non-target GOOS lacks one, that GOOS
is out of scope and would need its own noop-returning newOSCollector file — not required for this phase.
</design_note>
