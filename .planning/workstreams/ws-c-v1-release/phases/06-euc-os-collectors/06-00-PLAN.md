Create the single, shared, build-tag-FREE match helpers in internal/collector/euc/netcommon.go that
the Linux (06-01), Windows (06-02), and darwin (06-03) collectors all consume. This plan is the ONLY
owner of netcommon.go — extracting it into its own Wave-0 plan removes the shared-file ownership
hazard that would otherwise occur if 06-01/06-02/06-03 each tried to create or fall back to creating
it in parallel (duplicate-declaration / parallel-build failure).

imports NOTHING platform-specific, and therefore cannot drag a heavy platform dep (cilium/ebpf,
golang-etw) into any build graph. The three Wave-1 platform plans depend on this plan and simply
consume matchHost / matchPort / the local-inference-port helper from package euc.

Purpose: SC-4 (config-driven watch-list matching shared by all platforms), and the dependency-
isolation guarantee (Pitfall 6) for the tag-free helper layer. R-64, R-65.

Output: internal/collector/euc/netcommon.go + netcommon_test.go. NO new module (pure stdlib). NO
change to euc.go, euc_noop.go, Observation, Config, or fanOut.

Scope fence: helpers ONLY — no capture mechanism, no platform import, no build tag. Any eBPF/ETW/
gopsutil capture logic belongs in the Wave-1 platform plans, not here.
