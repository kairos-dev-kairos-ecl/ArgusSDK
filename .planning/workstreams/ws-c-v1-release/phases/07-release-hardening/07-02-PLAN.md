<objective>
Implement bounded config hot-reload on SIGHUP, resolving the agent.go:157 TODO and delivering the README "hot-reloadable" EUC promise (07-CONTEXT LOCKED "Config hot-reload" + ROADMAP Phase 7 Success Criterion 2 / R-72).

LOCKED scope (do not exceed): SIGHUP re-reads the config file and applies ONLY:
  (a) the EUC watch list — euc.Config.AIEndpoints + LocalInferencePorts;
  (b) the log level — logging.level.
Everything else (outputs/connectors/listeners/buffer/tls/ingest) is restart-required and must be documented as such. SIGHUP must never rebuild connectors or rebind listeners.

Seam constraint (LOCKED): the EUC collectors (Phase 6 impls behind euc.OSCollector) must accept an updated watch list at runtime WITHOUT changing the OSCollector / Observation / fanOut contract. Coordinate via a shared atomic config value that the Collector exposes through an exported UpdateWatchList method; the OS impls read the current watch list from that shared value rather than from their construction-time copy.

Log-level wiring (LOCKED — the live-effect requirement): today cmd/argus-agent/main.go buildLogger() builds a zap.AtomicLevel locally (as zapCfg.Level) and DISCARDS it after Build(), so nothing can mutate the running logger's level. This plan MUST fix that: buildLogger RETURNS the zap.AtomicLevel it used as zapCfg.Level (the one wired into the live zap core); main.go passes BOTH cfgFile AND that AtomicLevel into the agent; the agent's SIGHUP reload calls SetLevel on THAT SAME instance. The acceptance must prove the live effect — after a reload to debug, the running logger actually enables DebugLevel — not merely that a struct field changed.

Failure safety (LOCKED): a bad reload (file unreadable, malformed YAML, unmarshal error) is rejected — log a warning, keep the previous watch list and log level, never crash.

Constraints (binding):
- This plan touches internal/agent/* , internal/collector/euc/euc.go (+ tests), and cmd/argus-agent/main.go. It is file-disjoint from 07-01 (ocsf), 07-03 (docs/config/build), and 07-04 (ci) → Wave 1.
- main.go change is bounded to the log-level seam: buildLogger returns the AtomicLevel; main wires cfgFile + the AtomicLevel into the agent. Do not otherwise change main.go behaviour (flags, viper read, defaults preserved).
- Do NOT change the euc.OSCollector interface, the Observation struct, or the fanOut method signature. The euc_noop / linux / windows / darwin impls must still compile unchanged (they may, at most, read a shared watch-list value passed at construction — but their method set stays identical).
- Windows dev box: tests must not require sending a real SIGHUP if that is awkward on Windows — test reloadConfig directly (call the reload function with a new in-memory/temp-file config) AND, separately, assert the SIGHUP wiring exists. Use config written to a temp file + viper/yaml re-read; Close file handles before delete.

Purpose: operators tune the Shadow-AI watch list and verbosity in production without bouncing the agent and dropping buffered signals; the README already promises a hot-reloadable EUC list.
Output: a SIGHUP reload path with a bounded, tested, fail-safe apply; an EUC watch-list update seam; a live, mutable log level wired end-to-end from main.go.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| On-disk config file (operator-writable) → live agent state on SIGHUP | A reload reads attacker-or-operator-influenced YAML and mutates the running watch list + log verbosity |
| SIGHUP signal source → reload trigger | Any local process able to signal the agent can trigger a re-read |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-04 | Denial of Service | malformed config on SIGHUP crashes agent | mitigate | reloadConfig validates (read+unmarshal) before applying; on any error it logs and returns leaving prior state intact — never panics, never partially applies |
| T-07-05 | Elevation / scope creep | SIGHUP reconfigures outputs/listeners (unintended live changes) | mitigate | Apply is bounded to EUC watch list + log level only; outputs/connectors/listeners explicitly untouched and documented restart-required |
| T-07-06 | Tampering (race) | concurrent watch-list read by OS impl vs reload write | mitigate | Watch list held in atomic.Value/RWMutex with copy-in/copy-out; concurrent -race test in CI |
| T-07-07 | Information Disclosure | log level flipped to debug leaks verbose data | accept | Operator-controlled by design (same trust as config-file write); bounded to the agent's own logs |
| T-07-SC | Tampering | package installs | accept | No new dependencies — uses stdlib os/signal/syscall, existing viper + zap.AtomicLevel |
</threat_model>
