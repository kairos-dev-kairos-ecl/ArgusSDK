<objective>
Complete the connector framework with two missing production requirements:

1. Shared TLS enforcement helper (internal/connector/tls.go): a single function all connectors call to build a *tls.Config with TLS 1.3 minimum. Eliminates per-connector TLS drift and prevents InsecureSkipVerify.

2. Config-driven connector selection with hot-reload (internal/connector/config.go): YAML-tagged ConnectorConfig structs and a Watcher that uses fsnotify (already in go.mod via viper) to reload connector config without restart.

The Connector interface in connector.go is already complete (Connect/Send/Health/Close + ConnectorRegistry + Dispatcher). This plan adds the missing framework utilities without modifying the interface.

Purpose: Every connector in Wave 2 must call NewTLSConfig from this plan instead of building their own tls.Config. The Watcher gives operators hot-reload (SC-8).
Output: tls.go, config.go, config_test.go in internal/connector/.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| YAML config file | Operator-writable config on disk; parsed and re-applied on file change |
| TLS cert/key files | PEM files read from disk; passed directly to crypto/tls |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03 | Tampering | ConnectorConfig YAML hot-reload | mitigate | Watcher calls LoadConnectorsConfig and validates parse before invoking onChange; malformed YAML does not apply partial config |
| T-02-04 | Elevation of Privilege | TLS InsecureSkipVerify | mitigate | NewTLSConfig never sets InsecureSkipVerify; enforced by test TestNewTLSConfig_NoInsecureSkipVerify |
| T-02-05 | Tampering | CA cert file | mitigate | AppendCertsFromPEM returns bool; false → error; caller cannot proceed with untrusted pool |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No new external packages added; fsnotify already in go.mod as transitive dependency |
</threat_model>
