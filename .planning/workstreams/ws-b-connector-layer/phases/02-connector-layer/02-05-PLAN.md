<objective>
Create the Elastic/OpenSearch connector from scratch (no scaffold exists) using stdlib net/http and the Elasticsearch /_bulk API.

No elastic client library is used — the Bulk API is straightforward NDJSON over HTTP, and avoiding the go-elasticsearch SDK keeps the dependency surface minimal and avoids version-pinning complexity with OpenSearch. The connector works against both Elasticsearch 8.x and OpenSearch 2.x (identical /_bulk API surface).

OCSF→ECS field mapping is done inline in the connector (not a separate mapper package). ECS is the target schema for Elastic/OpenSearch; OCSF is the intermediate. The mapping is bounded: 7 key ECS fields derived from the OCSF Event struct.

Purpose: Elastic/OpenSearch is the third priority connector (SC-5). Creates the third connector, completing Wave 2.
Output: elastic/connector.go and connector_test.go created; no new go.mod dependencies required.
</objective>

Elasticsearch /_bulk API:
  POST <endpoint>/_bulk
  Content-Type: application/x-ndjson
  Body: alternating lines — action line + document line, each terminated by "\n"
  Action line: {"index":{"_index":"<index>"}}
  Document line: <JSON document>
  Response: {"took":N,"errors":false,"items":[...]} — check errors field
  Auth: Authorization: ApiKey <base64(id:key)>

ECS (Elastic Common Schema) field mapping from OCSF:
  @timestamp  ← time.UnixMilli(ev.Time).UTC().Format(time.RFC3339Nano)
  event.code  ← strconv.Itoa(int(ev.ClassUID))
  event.category ← []string{ev.CategoryName}
  event.kind  ← "event" (always for non-alert signals)
  event.severity ← severityIDToECSText(ev.SeverityID)  // "informational"|"low"|"medium"|"high"|"critical"
  agent.name  ← ev.Metadata.Product.VendorName
  event.module ← ev.Metadata.Product.Name
  event.uid   ← ev.Metadata.UID
