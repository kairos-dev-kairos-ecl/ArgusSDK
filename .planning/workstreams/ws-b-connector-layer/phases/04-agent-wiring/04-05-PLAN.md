## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| config → factory/registry | Operator config drives which connectors and credentials are built |
| collectors → ingest loop | Collected signals (incl. untrusted local SDK input) flow into dispatch/buffer |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-15 | Denial of service | ingest loop / dispatcher queue | mitigate | On Enqueue failure (queue full) the batch is Written to the WAL buffer, not dropped; bounded queue capacity preserves backpressure |
| T-04-16 | Repudiation | drain on shutdown | mitigate | stop() Flushes with the real drain (no F5 nil path); failed delivery returns non-nil error so buffer retries — no silent loss |
| T-04-17 | Elevation of privilege | registration | accept | Local registrar derives InstanceID deterministically; remote registration deferred (locked decision 8); no live-XDR trust introduced |
| T-04-18 | Tampering | per-target OCSF | mitigate | OCSF and non-OCSF targets receive SEPARATE DispatchJobs with the correct per-group UseOCSF; argusxdr is always in the non-OCSF group (locked decisions 2/5) — no operator override downgrades the XDR proto path |
</threat_model>
