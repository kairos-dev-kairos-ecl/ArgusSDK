<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| buf remote plugins → local filesystem | buf downloads remote plugin executables and runs them against proto source; executes arbitrary code from buf.build registry |
| go get (network) → go.mod | Fetches protobuf/grpc packages from pkg.go.dev; supply chain risk |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | buf remote plugins (protocolbuffers/go, grpc/go) | accept | buf.build is the official registry maintained by Buf Inc; plugins are the canonical protoc-gen-go and protoc-gen-go-grpc maintained by Google/grpc teams; no third-party plugins used |
| T-02-02 | Tampering | google.golang.org/protobuf and google.golang.org/grpc in go.mod | mitigate | Both are Google-maintained official packages on pkg.go.dev with known checksums; go.sum pins exact hashes after `go mod tidy` |
| T-02-03 | Denial of Service | buf generate fails silently | mitigate | make proto has explicit `command -v buf` guard that fails with exit code 1 and error message |
| T-02-SC | Tampering | go get network fetches | mitigate | go.sum checksum database verification enforced by Go toolchain; GONOSUMCHECK not set |
</threat_model>
