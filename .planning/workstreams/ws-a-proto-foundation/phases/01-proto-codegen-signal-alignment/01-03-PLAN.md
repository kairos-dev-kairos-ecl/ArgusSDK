type Signal struct {
  // Identity
  SignalID     string
  TraceID      string
  SpanID       string
  ParentSpanID string

  // Classification
  Layer    Layer       // Layer is int32-based type
  Category string
  Severity Severity    // Severity is int32-based type

  // Source metadata
  AppID      string
  AppVersion string
  SDKVersion string
  Env        string

  // Temporal
  Timestamp  time.Time
  DurationMS float32

  // Payload
  ContextJSON []byte

  // Relationships
  SessionID      string
  ConversationID string
  UserID         string
}

type Batch struct {
  AppID   string
  Env     string
  Signals []Signal
}

<!-- SDKSignal proto fields (from Plan 01-01 — generated into gen/go/sdk/v1/) -->
Proto SDKSignal field → Signal struct mapping (all 14 fields):
  p.SignalId      → s.SignalID
  p.TraceId       → s.TraceID
  p.SpanId        → s.SpanID
  p.ParentSpanId  → s.ParentSpanID
  p.Layer         → s.Layer    (cast: Layer(p.Layer))
  p.Category      → s.Category
  p.Severity      → s.Severity (cast: Severity(p.Severity))
  p.Timestamp     → s.Timestamp (convert: p.Timestamp.AsTime())
  p.DurationMs    → s.DurationMS
  p.ContextJson   → s.ContextJSON
  p.SessionId     → s.SessionID
  p.ConversationId → s.ConversationID
  p.UserId        → s.UserID
  p.AppVersion    → s.AppVersion

Note: AppID, Env, SDKVersion come from SignalBatch — NOT from SDKSignal. These are NOT set by FromProto.
Note: p.Timestamp may be nil (proto3 optional semantics for message fields). Guard with: if p.Timestamp != nil { s.Timestamp = p.Timestamp.AsTime() }

<!-- ToProto mapping: Batch → SignalBatch -->
SignalBatch fields:
  BatchId    ← batchID parameter
  AppId      ← b.AppID
  Env        ← b.Env
  Signals    ← each b.Signals[i] converted via signal.ToProtoSignal()
  SdkVersion ← sdkVersion parameter

SDKSignal (from Signal) — reverse of FromProto, all 14 fields.
  For timestamp: timestamppb.New(s.Timestamp)
  This requires import "google.golang.org/protobuf/types/known/timestamppb"

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SDKSignal.user_id → Signal.UserID | UserID passes through as raw string; SDK lib is responsible for hashing before send; agent trusts SDK lib on local hop |
| SDKSignal.context_json → Signal.ContextJSON | Raw bytes passed through; agent validates JSON structure per-layer in WS-E, not in FromProto |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | Signal.UserID in FromProto | accept | UserID hashing is enforced at SDK lib level (documented in proto comment field 13); FromProto is agent-side and receives already-hashed values; adding a second hash in FromProto would break the contract |
| T-03-02 | Tampering | ContextJSON bytes passed through without validation | accept | Validation is WS-E responsibility (per CONTEXT.md scope boundary — "Signal Validation" is E2); FromProto is a pure field mapper, not a validator |
| T-03-03 | Denial of Service | nil Timestamp causing panic in FromProto | mitigate | Explicit nil guard on p.Timestamp before calling .AsTime() — tested by Test 2 in behavior block |
| T-03-SC | Tampering | No package manager installs in this plan | accept | No new packages — google.golang.org/protobuf was added in Plan 01-02; this plan only adds an import, not a new dependency |
</threat_model>
